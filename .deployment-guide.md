# デプロイガイド

## Vercel

### CLIでデプロイ

```bash
# Vercel CLIをインストール
npm install -g vercel

# プロジェクトをデプロイ
vercel

# 本番環境へデプロイ
vercel --prod
```

### Webダッシュボードでデプロイ

1. https://vercel.com にアクセス
2. 「New Project」をクリック
3. GitHubリポジトリをインポート
4. ビルド設定は自動検出されます（vercel.jsonがあるため）
5. 「Deploy」をクリック

## Cloudflare Pages

### Wrangler CLIでデプロイ

```bash
# ビルド
npm run build

# デプロイ
npx wrangler pages deploy dist --project-name=ai-meeting-notes
```

### Webダッシュボードでデプロイ

1. https://dash.cloudflare.com にアクセス
2. 「Pages」→「Create a project」
3. GitHubリポジトリを接続
4. ビルド設定:
   - Build command: `npm run build`
   - Build output directory: `dist`
   - Environment variables: 不要
5. 「Save and Deploy」をクリック

## 環境変数

このアプリは完全にクライアントサイドで動作するため、環境変数の設定は不要です。  
APIキーはユーザーがブラウザで入力し、localStorageに保存されます。

## カスタムドメイン

デプロイ後、各プラットフォームのダッシュボードからカスタムドメインを設定できます。

### Vercel
1. プロジェクト設定 → Domains
2. ドメインを追加してDNS設定

### Cloudflare Pages
1. プロジェクト設定 → Custom domains
2. ドメインを追加（Cloudflareで管理している場合は自動設定）

## CI/CD

両プラットフォームとも、GitHubリポジトリと連携すると自動デプロイが有効になります。

- `main`ブランチへのpush → 本番環境に自動デプロイ
- その他のブランチ → プレビュー環境に自動デプロイ
